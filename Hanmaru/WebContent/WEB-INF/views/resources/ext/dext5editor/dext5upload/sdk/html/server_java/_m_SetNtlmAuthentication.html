DEXT5 Upload :: SetNtlmAuthentication

void SetNtlmAuthentication(String domain, String username, String password)

파일을 업로드 하려는 위치가 계정이 있는 네트워크 드라이브이거나,
다운로드 하려는 파일의 위치가 계정이 있는 네트워크 드라이브일 경우
네트워크 드라이브의 인증 정보를 설정해야 합니다.
JAVA에서는 SMB 프로토콜을 사용하며, 역슬래시(\)가아닌 슬래시(/)로 경로설정이 되어야합니다.

return value

없음.

parameters

domain  Network Drive 접속 Domain을 의미합니다.
username  Network Drive 인증 User Name를 의미합니다.
password  Network Drive 인증 Password를 의미합니다.

remarks

1. SMB Protocol 연결 설정
SMB Protocol 사용을 위한 연결설정을 해야 합니다.
ex) upload.SetNtlmAuthentication("smb://SMB Domain 또는 IP", "ID", "Password");
2. Multipart Temp Path 설정
JAVA에서 Multpart Stream이 특정 폴더에 저장된 후 그 Stream을 실제 저장 폴더에 쓰고,
이 폴더는 Smb Protocol 경로설정이 불가하므로 해당 서버의 특정 Temp 경로를 설정해야 합니다.
이 설정을 하지 않으면 해당 폴더가 파일 업로드시 사용되는 Temp Path로 설정되어 연결Error 발생
ex) upload.SetMultipartTempPath("D:\\temp");
3. 파일 업로드시 사용되는 Temp Path 설정
파일 업로드시 사용되는 Temp Path를 SMB Protocol의 경로로 설정해야 합니다.
ex) upload.SetTempPath("smb://SMB Domain 또는 IP/temp ");
4. 파일이 업로드 될 최종 Path 설정
파일이 업로드 될 최종 Path를 SMB Protocol의 경로로 설정해야 합니다.
ex) upload.SetPhysicalPath("smb://SMB Domain 또는 IP/savePath");

sample code

Raonwiz.Dext5.UploadHandler upload = new Raonwiz.Dext5.UploadHandler();

upload.SetNtlmAuthentication("smb://SMB Domain 또는 IP", "ID", "Password");
upload.SetMultipartTempPath("D:\\temp");
upload.SetTempPath("smb://SMB Domain 또는 IP/temp ");
upload.SetPhysicalPath("smb://SMB Domain 또는 IP/savePath");

String result = upload.Process(request, response, event);
